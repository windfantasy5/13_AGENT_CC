# 知识库重置功能测试说明

## 测试环境
- 后端：http://127.0.0.1:8000
- 前端：http://localhost:3000
- 测试时间：2026-04-18

## 测试步骤

### 1. 访问知识库管理页面
1. 打开浏览器访问：http://localhost:3000
2. 登录系统（如果未登录）
3. 点击导航栏的"知识库管理"

### 2. 测试重置按钮
1. 在知识库管理页面，查看导航栏右侧
2. 应该能看到红色的"重置知识库"按钮
3. 点击"重置知识库"按钮

### 3. 测试确认对话框
1. 应该弹出确认对话框
2. 对话框标题：⚠️ 危险操作
3. 对话框内容包含：
   - 删除所有已上传的文档记录
   - 删除所有文档分块数据
   - 删除本地向量数据库
   - 此操作不可恢复
4. 有一个输入框，提示输入"确认重置"

### 4. 测试输入验证
1. 不输入任何内容，点击"确认重置"按钮
   - 预期：按钮应该是禁用状态（灰色）
2. 输入错误的文本（如"确认"），点击"确认重置"
   - 预期：按钮应该是禁用状态
3. 输入正确的文本"确认重置"
   - 预期：按钮变为可点击状态（红色）

### 5. 测试取消功能
1. 点击"取消"按钮
   - 预期：对话框关闭，不执行任何操作

### 6. 测试重置功能（实际执行）
**注意：此操作会删除所有数据，请确认可以执行**

1. 再次点击"重置知识库"按钮
2. 输入"确认重置"
3. 点击"确认重置"按钮
4. 等待操作完成
5. 预期结果：
   - 显示成功提示："知识库已重置！所有文档和向量数据已删除。"
   - 对话框自动关闭
   - 当前预览内容被清空

### 7. 验证后端数据
1. 检查向量数据库目录是否被删除：
   ```bash
   ls D:/00_source/30_Python312/13_AGENT_CC/chroma_db
   ```
   预期：目录不存在或为空

2. 检查MySQL数据库：
   ```sql
   SELECT COUNT(*) FROM documents;
   SELECT COUNT(*) FROM document_chunks;
   ```
   预期：两个表的记录数都为0
