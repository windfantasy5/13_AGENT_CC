- `backend/app/services/chat_service.py` - 添加流式方法

- `frontend/src/pages/Chat.tsx` - 使用流式响应

### 3. ✅ 实现流式回答和检索信息显示

**功能需求：**
1. 先流式显示检索到的信息
2. 然后流式显示大模型的答案
3. 全部显示完成后，只保留大模型答案，移除检索信息

**实现方案：**

#### 后端实现

**新增流式接口：** `POST /api/v1/chat/message/stream`

**流式响应格式（SSE）：**

```json
// 1. 用户消息确认
{
  "type": "user_message",
  "data": {
    "id": 123,
    "role": "user",
    "content": "用户问题",
    "created_at": "2026-04-15T10:00:00"
  }
}

// 2. RAG检索结果（临时显示）
{
  "type": "rag_context",
  "data": {
    "results": [
      {
        "content": "检索到的文档内容",
        "score": 0.85,
        "document_title": "文档标题"
      }
    ]
  }
}

// 3. 助手回复流式片段
{
  "type": "assistant_chunk",
  "data": {
    "content": "回答的一部分"
  }
}

// 4. 助手回复完成
{
  "type": "assistant_message",
  "data": {
    "id": 124,
    "role": "assistant",
    "content": "完整的回答",
    "tokens": 150,
    "model": "gpt-3.5-turbo",
    "created_at": "2026-04-15T10:00:05"
  }
}

// 5. 流结束标记
[DONE]
```

#### 前端实现

**流式处理逻辑：**

1. 发送用户消息，立即显示在界面
2. 接收 `rag_context` 类型消息，显示检索信息
3. 接收 `assistant_chunk` 类型消息，逐字显示AI回答
4. 接收 `assistant_message` 类型消息，更新最终消息ID
5. 接收 `[DONE]` 标记，移除RAG检索信息

**修改文件：**
- `backend/app/api/v1/chat.py`
- `backend/app/services/chat_service.py`
- `frontend/src/services/api.ts`
- `frontend/src/pages/Chat.tsx`

## 技术细节

### 后端流式实现

```python
@router.post("/message/stream")
async def send_message_stream(...):
    async def generate():
        # 1. 发送用户消息
        yield f"data: {json.dumps(user_msg_data)}\n\n"

        # 2. RAG检索并发送
        if use_rag:
            rag_results = await rag_service.search_knowledge(...)
            yield f"data: {json.dumps(rag_data)}\n\n"

        # 3. 流式调用LLM
        async for chunk in llm_service.chat_completion_stream(...):
            yield f"data: {json.dumps(chunk)}\n\n"

        # 4. 发送完成消息
        yield f"data: {json.dumps(final_msg)}\n\n"
        yield "data: [DONE]\n\n"

    return StreamingResponse(generate(), media_type="text/event-stream")
```

  const { done, value } = await reader.read();
  if (done) break;

  buffer += decoder.decode(value, { stream: true });

## 已知限制

1. **LLM流式支持**
   - 当前实现检查 `llm_service.chat_completion_stream` 是否存在
   - 如果不支持流式，会降级到非流式模式
   - 建议实现真正的LLM流式调用以获得最佳体验

2. **Token计数**
   - 流式模式下使用简单的字符长度估算tokens
   - 建议集成tiktoken等库进行精确计数

3. **错误恢复**
   - 流式传输中断时，前端会清理临时消息
   - 建议添加重试机制
