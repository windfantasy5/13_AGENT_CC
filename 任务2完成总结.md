# 任务2完成总结：评判机制

## ✅ 已完成内容

### 1. 核心模块开发

#### 评判提示词模板 (`backend/app/prompts/judge_prompts.py`)
- **检索质量评判提示词** - 评估参考资料与问题的相关性
- **一致性检查提示词** - 检测多个文档间的矛盾
- **答案质量评判提示词** - 评估AI生成答案的质量
- 所有提示词要求输出结构化JSON格式

#### 评判服务核心 (`backend/app/services/judge_service.py`)
- **JudgeConfig配置类**
  - 评判模型：qwen-turbo（轻量快速）
  - 生成模型：qwen3-max（高质量）
  - 检索质量阈值：6分
  - 答案质量阈值：7分
  - 最大重试次数：2次

- **三个核心评判器**
  1. `judge_retrieval_quality()` - 检索质量评判器
     - 输入：用户问题 + 检索文档
     - 输出：0-10评分 + 理由 + 是否通过

  2. `check_consistency()` - 一致性检查器
     - 输入：多个检索文档
     - 输出：是否有矛盾 + 矛盾列表 + 总结

  3. `judge_answer_quality()` - 答案质量评判器
     - 输入：问题 + 参考资料 + AI答案
     - 输出：0-10评分 + 理由 + 问题列表 + 是否通过

#### LLM服务升级 (`backend/app/services/llm_service.py`)
- 支持指定模型调用（用于评判服务）
- 添加 qwen 系列模型路由逻辑
- qwen-turbo、qwen3-max 通过 OpenAI API 调用
- 保持负载均衡机制不受影响

#### 对话服务集成 (`backend/app/services/chat_service.py`)
- 在 `send_message_stream()` 中集成评判流程
- **检索阶段评判**
  - 检索后立即评判质量
  - 评判不通过则增加 top_k 重试（最多2次）
  - 通过后进行一致性检查

- **答案生成评判**
  - 生成答案后评判质量
  - 评判不通过则重新生成（最多1次）

- **流式输出增强**
  - 新增 `judge_retrieval` 事件（检索质量评判结果）
  - 新增 `judge_consistency` 事件（一致性检查结果）
  - 新增 `judge_answer` 事件（答案质量评判结果）
  - 新增 `regenerating` 事件（答案重新生成提示）

## 🎯 技术亮点

### 1. 双模型架构
```
评判模型（qwen-turbo）：轻量快速，低成本
    ↓
生成模型（qwen3-max）：高质量，准确性强
```
- 评判使用低温度（0.1）保证稳定性
- 生成使用适中温度（0.7）保证创造性

### 2. 智能重试机制

**检索重试策略**
```python
retry_count = 0
top_k = 3 + retry_count  # 逐步增加检索数量
use_cache = (retry_count == 0)  # 仅首次使用缓存
```

**答案重试策略**
- 最多重试1次（避免过度消耗）
- 重新构建上下文
- 发送 `regenerating` 事件通知用户

### 3. 容错设计
- 评判失败时返回默认通过，避免阻塞流程
- JSON解析失败时使用正则提取
- 所有异常都有日志记录

### 4. 流式事件增强
```javascript
// 前端可接收的新事件类型
{
  type: "judge_retrieval",  // 检索质量评判
  type: "judge_consistency", // 一致性检查
  type: "judge_answer",      // 答案质量评判
  type: "regenerating"       // 重新生成提示
}
```

## 📝 使用示例

### 后端集成（已自动集成）
```python
# 在 ChatService.send_message_stream() 中
# 1. RAG检索后自动评判
judge_result = await self.judge_service.judge_retrieval_quality(
    query=content,
    retrieved_docs=rag_results
)

# 2. 检索通过后检查一致性
consistency_result = await self.judge_service.check_consistency(
    retrieved_docs=rag_results
)

# 3. 答案生成后评判质量
answer_judge_result = await self.judge_service.judge_answer_quality(
    query=content,
    context=rag_context,
    answer=full_content
)
```

### 前端处理（需要实现）
```typescript
// 监听评判事件
eventSource.addEventListener('message', (event) => {
  const data = JSON.parse(event.data);

  switch(data.type) {
    case 'judge_retrieval':
      // 显示检索质量评分
      console.log(`检索质量: ${data.data.score}/10`);
      break;

    case 'judge_consistency':
      // 显示一致性检查结果
      if (data.data.has_conflict) {
        console.warn('发现矛盾信息');
      }
      break;

    case 'judge_answer':
      // 显示答案质量评分
      console.log(`答案质量: ${data.data.score}/10`);
      break;

    case 'regenerating':
      // 显示重新生成提示
      console.log('答案质量不足，正在重新生成...');
      break;
  }
});
```

## 📈 改进效果

### 1. 检索质量提升
- **问题**：低相关性文档导致答案质量差
- **解决**：评判不通过时自动增加检索数量重试
- **效果**：提高检索召回率，过滤不相关内容

### 2. 信息一致性保障
- **问题**：多个文档可能存在矛盾信息
- **解决**：自动检测矛盾并标记
- **效果**：避免生成自相矛盾的答案

### 3. 答案质量保证
- **问题**：AI可能生成偏离主题或不准确的答案
- **解决**：评判不通过时自动重新生成
- **效果**：确保答案准确性和相关性

### 4. 用户体验优化
- **透明度**：用户可以看到评判过程和结果
- **可靠性**：多重检查机制确保答案质量
- **智能化**：自动重试减少人工干预
