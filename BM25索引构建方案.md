## 二、BM25索引构建策略

### 2.1 索引构建时机

#### 方案A：同步构建（推荐）
**触发时机**：文档向量化完成后立即构建BM25索引

**优点**：
- 索引实时更新
- 数据一致性强
- 用户体验好

**实现**：
```python
# 在 document_service.py 的 confirm_chunks 方法中
async def confirm_chunks(...):
    # 1. 保存分块到数据库
    # 2. 向量化并存储
    # 3. 构建BM25索引（新增）
    await self._update_bm25_index(document_id, db)
```

#### 方案B：异步构建
**触发时机**：后台定时任务批量构建

**优点**：
- 不阻塞用户操作
- 批量处理效率高

**缺点**：
- 索引延迟
- 实现复杂

### 2.2 索引更新策略

#### 增量更新方案（推荐）
```python
# 新增文档
add_document(chunk_id, tokens) → 追加到索引

# 删除文档
delete_document(chunk_id) → 从索引移除

# 更新文档
update_document(chunk_id, tokens) → 删除 + 新增
```

**技术实现**：
- 使用 `BM25Plus` 替代 `BM25Okapi`（支持增量更新）
- 或者维护文档-token映射，重建时排除已删除文档

#### 全量重建方案（当前实现）
```python
# 定期重建整个索引
rebuild_index() → 从数据库读取所有文档 → 重新构建
```

**适用场景**：
- 文档数量较少（<10万）
- 更新频率低
- 对实时性要求不高

## 三、高质量分词方案

### 3.1 分词优化

#### 当前实现（基础版）
```python
def tokenize(text):
    tokens = jieba.lcut(text)
    tokens = [t.strip() for t in tokens if len(t.strip()) > 1]
    return tokens
```

**问题**：
- 包含大量无意义词（的、了、是）
- 未区分词性
- 未处理专业术语

#### 优化方案（专业版）
```python
import jieba.posseg as pseg
from pathlib import Path

class AdvancedTokenizer:
    def __init__(self):
        # 加载停用词
        self.stopwords = self._load_stopwords()
        # 加载自定义词典
        self._load_custom_dict()

    def tokenize(self, text: str) -> List[str]:
        # 1. 词性标注分词
        words = pseg.lcut(text)

        # 2. 过滤规则
        tokens = []
        for word, flag in words:
            # 保留名词(n)、动词(v)、形容词(a)、专有名词(nr/ns/nt)
            if flag.startswith(('n', 'v', 'a')) or flag in ['nr', 'ns', 'nt']:
                # 过滤停用词
                if word not in self.stopwords and len(word) > 1:
                    tokens.append(word.lower())

        return tokens

    def _load_stopwords(self) -> set:
        # 加载中文停用词表
        stopwords_file = Path("data/stopwords.txt")
        if stopwords_file.exists():
            return set(stopwords_file.read_text(encoding='utf-8').splitlines())
        return set()

    def _load_custom_dict(self):
        # 加载领域专业词典
        dict_file = Path("data/custom_dict.txt")
        if dict_file.exists():
            jieba.load_userdict(str(dict_file))
```

## 四、索引存储方案

### 4.1 Redis存储结构

#### 方案A：序列化存储（当前实现）
```
Key: bm25:index
Value: pickle(BM25Okapi对象)

Key: bm25:corpus_ids
Value: pickle([chunk_id1, chunk_id2, ...])
```

**优点**：简单直接
**缺点**：不支持增量更新

#### 方案B：结构化存储（推荐）
```
# 文档-token映射
Key: bm25:doc:{chunk_id}
Value: ["token1", "token2", ...]

# Token-文档倒排索引
Key: bm25:token:{token}
Value: {chunk_id1: tf1, chunk_id2: tf2, ...}

# 文档频率
Key: bm25:df:{token}
Value: document_frequency

# 文档长度
Key: bm25:doclen:{chunk_id}
Value: document_length

# 平均文档长度
Key: bm25:avgdl
Value: average_document_length

# 文档总数
Key: bm25:total_docs
Value: total_document_count
```

**优点**：
- 支持增量更新
- 可以单独删除文档
- 便于调试和监控

**缺点**：
- 存储空间较大
- 实现复杂

### 4.2 混合存储方案（最佳实践）
```
# 核心索引：序列化存储（快速加载）
Key: bm25:index
Value: pickle(BM25对象)

# 元数据：结构化存储（支持增量）
Key: bm25:meta:corpus_ids
Value: [chunk_id1, chunk_id2, ...]

Key: bm25:meta:last_update
Value: timestamp

Key: bm25:meta:version
Value: version_number
```

## 五、索引构建流程

### 5.1 完整流程

```
1. 文档上传
   ↓
2. 文本提取（PDF/Word/TXT）
   ↓
3. 智能分段（SmartTextSplitter）
   ↓
4. 用户确认分段参数
   ↓
5. 并行处理：
   ├─ 向量化（text-embedding-v4）→ ChromaDB
   └─ 分词（jieba + 停用词）→ BM25索引 → Redis
   ↓
6. 更新数据库状态
   ↓
7. 返回成功
```

### 5.2 代码实现位置

```python
# backend/app/services/document_service.py
async def confirm_chunks(...):
    # 现有代码：向量化
    vector_ids = self.vector_store.add_chunks(...)

    # 新增代码：构建BM25索引
    await self._update_bm25_index(document_id, db)

async def _update_bm25_index(self, document_id: int, db: AsyncSession):
    """更新BM25索引"""
    from app.core.bm25_retriever import BM25Retriever
    retriever = BM25Retriever()

    # 获取该文档的所有分块
    stmt = select(DocumentChunk).where(
        DocumentChunk.document_id == document_id,
        DocumentChunk.vector_id.isnot(None)
    )
    result = await db.execute(stmt)
    chunks = result.scalars().all()

    # 增量添加到索引
    for chunk in chunks:
        await retriever.add_document_incremental(chunk.id, chunk.content)
```

### 6.1 索引构建优化

#### 批量处理

# 批量构建索引（每100个文档一批）
async def build_index_batch(chunks: List[DocumentChunk], batch_size=100):
    for i in range(0, len(chunks), batch_size):
        batch = chunks[i:i+batch_size]
        # 批量分词
        corpus = [tokenize(chunk.content) for chunk in batch]
        # 批量构建
        bm25_index.add_batch(corpus)

import asyncio

# 向量化和BM25索引并行构建
async def parallel_indexing(chunks):

## 八、最佳实践建议

### 8.1 小规模系统（<1万文档）
- 使用全量重建策略
- 文档上传后手动触发索引构建
- Redis序列化存储
- 基础分词（jieba + 长度过滤）

### 8.2 中等规模系统（1万-10万文档）
- 使用增量更新策略
- 文档上传后自动构建索引
- Redis混合存储
- 优化分词（停用词 + 词性过滤）

### 8.3 大规模系统（>10万文档）
- 使用专业搜索引擎（Elasticsearch）
- 分布式索引
- 多级缓存
- 高级分词（同义词 + 实体识别）
