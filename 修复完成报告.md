## 一、修复的问题

### 问题1：向量数据库集合重新创建导致数据丢失

**根本原因**：
- ChromaDB使用相对路径 `./chroma_db`
- 不同工作目录启动后端服务，导致创建多个不同位置的向量数据库
- 每次重启可能创建新集合，旧数据无法访问

**修复方案**：
1. 在 `settings.py` 中计算项目根目录（`PROJECT_ROOT`）
2. 新增 `chroma_db_abs_path` 属性，返回基于项目根的绝对路径
3. 修改 `vector_store.py` 使用绝对路径初始化ChromaDB
4. 同时修复上传目录 `UPLOAD_DIR` 也使用绝对路径

**修改文件**：
- `backend/app/config/settings.py`
- `backend/app/core/vector_store.py`
- `backend/app/services/document_service.py`

**效果**：
- 无论从哪个目录启动服务，向量数据库路径始终一致
- 避免数据丢失和集合重复创建问题
- 日志中会显示实际使用的绝对路径和已有向量数

## 四、可能的问题

### 1. 依赖未安装

**问题**：启动后端时报错 `ModuleNotFoundError: No module named 'pymupdf4llm'` 或 `html2text`

**解决**：
```bash
cd backend
pip install pymupdf4llm html2text
```

---

### 2. 旧向量数据无法访问

**问题**：修复后仍然检索不到之前上传的文档

**原因**：旧数据在不同路径的向量库中

**解决方案A（推荐）**：重新上传文档
1. 删除所有文档记录
2. 重新上传文档到统一路径的向量库

**解决方案B**：迁移旧数据
1. 找到旧的 `chroma_db` 目录
2. 复制到新的统一路径：`D:\00_source\30_Python312\13_AGENT_CC\chroma_db`
3. 重启服务

---

### 3. 网页抓取失败

**问题**：预览网页时报错"网页请求失败"或"网页处理失败"

**可能原因**：
- 网站需要登录或有反爬虫机制
- 网络连接问题
- URL格式错误

**解决**：
- 确认URL可以在浏览器中正常访问
- 尝试其他公开网页
- 检查网络连接

---

### 4. PDF转Markdown失败

**问题**：日志显示"PDF转Markdown失败，降级到基础提取"

**可能原因**：
- PDF文件损坏
- PDF格式特殊
- pymupdf4llm版本问题

**解决**：
- 系统会自动降级到PyPDF2提取，不影响使用
- 如需高级功能，尝试更新依赖：`pip install --upgrade pymupdf4llm PyMuPDF`
